8. What are some performance optimizations in spring boot micro services ?
		Be ready to discuss about 
			
			DB Layer - Indexes on frequent queries, 
					   avoid select * - get only required columns,
					   avoid N+1 quries
					   Use Pagination
					   Maintain Hikari CP connections - tune pool size
					   Hibernate caching (first level and second level)
					   Read - write DBs [ Write to primary DB and read from child DBs]
					   
			Service Layer: 1. Use Caching for idempotent request
						   2. Async processing - use reactive programming when needed
						   3. Use Projections instead of getting whole entity
						   4. Use circuit breakers and fallback mechanisms
						   5. Use DTOs instead of Entitys
						   6. Thread Pool Tuning
									props --> server.tomcat.threads.max=200
			
			API Gateway : 1. Load balancer and horizontal scaling.
						  2. Centralized security and authentication mechanisms.
						  3. Rate limiting and Routing
			
			Metrics     : 1. First identify where slowness is there using logging.
						  2. Optimize accordingly.
						  3. JVM level tuning
